## вңЁ Why this exists

Managing a WhatsApp group with 30+ members sounds easy until you're the one responsible for it. Spam links, forwarded chains, walls of ALL CAPS, random media floods вҖ” and someone always finds a way to break the rules.

So instead of moderating manually at 3 AM, I built a bot that does it automatically, consistently, and with a little bit of attitude.

---

## рҹҡҖ Features

### рҹӣЎпёҸ Smart Moderation Engine
- рҹҡ« **Blocks links** (http / https / www)
- рҹ–јпёҸ **Blocks all media** вҖ” images, videos, stickers, GIFs, voice notes, documents
- вңҚпёҸ **300-character limit** on text messages
- рҹ“Ё **Blocks forwarded messages**
- рҹ”Ҡ **Blocks ALL-CAPS spam**
- рҹ§© **Pluggable rule system** вҖ” add a new rule in ~2 minutes

### вҡ–пёҸ Progressive Discipline
- Strike system with auto-warnings
- **3 strikes вҶ’ automatic removal** from the group
- Admins & the group owner are always exempt

### рҹҺ« Permission Passes
- Grant a member a temporary bypass: `allow @user`
- **3 messages within 5 minutes**, then rules snap back on

### рҹ”Ҙ AI Personality (Powered by Groq)
The bot isn't just a filter вҖ” it talks, and its tone depends on **who** it's talking to:

| Who | Behavior |
|-----|----------|
| рҹ‘‘ **The Big Boss** | Total yes-man вҖ” loyal, obedient, agrees with everything |
| рҹӨқ **Co-Admin** | Respectful, helpful, addressed by title |
| рҹ”Ҙ **Members** | Savage, brutal roasts вҖ” refuses to obey anyone but the boss |

### рҹҰҫ From Chatbot вҶ’ Agent
Mention the bot or reply to it, and it **executes real commands**:
- `mention Arsalan and say good morning` вҶ’ really tags & messages him
- `roast @user` вҶ’ savage on-demand roast
- `announce: meeting at 5pm` вҶ’ group broadcast
- `tag all` вҶ’ mentions every member

### рҹ“ҹ Production-Grade Logging
Timestamped, color-coded, leveled logs (`INFO`, `OK`, `WARN`, `EVENT`, `ENFORCE`) that look like a real backend service.

## вҡЎ Quick Start

```bash
# 1. Clone
git clone https://github.com/wasiqtanveer/whatsapp-ai-moderator.git
cd whatsapp-ai-moderator

# 2. Install
npm install

# 3. Configure
cp .env.example .env   # then fill in your values

# 4. Run
npm start
```

Then **scan the QR code** in your terminal (WhatsApp вҶ’ Linked Devices вҶ’ Link a Device) and the bot goes live.

> рҹ’Ў On first run, leave `GROUP_ID` empty вҖ” the bot prints incoming chat IDs so you can find your group's ID (ends in `@g.us`), then paste it into `.env`.

## вҡҷпёҸ Configuration

All settings live in `.env` (see [`.env.example`](.env.example)):

| Variable | Purpose |
|----------|---------|
| `GROUP_ID` | The group the bot moderates |
| `GROQ_API_KEY` | Your free [Groq](https://console.groq.com/keys) API key |
| `BOSS_IDS` | The big boss вҖ” bot's loyal yes-man |
| `COADMIN_IDS` / `COADMIN_TITLE` | Co-admins it respects |
| `BOT_IDS` | The bot's own ID (so @mentions are recognized) |
| `BOT_KEYWORD` | Optional keyword trigger (empty = mention/reply only) |

Tweak rules in [`rules.js`](rules.js) and limits in [`config.js`](config.js).

## рҹ§© Engineering Highlights

- **Pluggable rule engine** вҖ” rules are self-contained objects, trivially extensible.
- **Graceful AI fallback** вҖ” if Groq fails, the bot silently uses template messages. The AI is the *fun* part; reliability comes first.
- **Cross-format ID matching** вҖ” handles WhatsApp's `@lid` vs `@c.us` ID quirk that silently breaks naive admin checks.
- **Role-based AI personas** вҖ” one model, three personalities, via prompt engineering.
- **Agentic command layer** вҖ” natural-language instructions parsed into structured JSON actions, then executed.
